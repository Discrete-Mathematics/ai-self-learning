# 七. 扩展补充：类模板、函数模板、及其他

## 1. static

### 1.1. 类与对象的内存分配

![image-20250529200605803](c++面向对象高级编程上.assets/image-20250529200605803-174852036696067-174852037720569.png)

一个类可以创建多个对象。每当创建一个对象时，系统会为该对象的**非静态数据成员**分配独立的内存空间；而**静态成员**和**成员函数**则在整个类中只存在一份，供所有对象共享。当对象调用成员函数时，编译器会**隐式地将该对象的地址作为 `this` 指针**传入函数内部。成员函数通过 `this` 指针来访问当前对象的成员，从而区分不同的对象实例。

### 1.2. static成员

![image-20250529202727199](c++面向对象高级编程上.assets/image-20250529202727199-174852164818171.png)

**要点**: 

1. static成员只能在类内声明, 不能在类中初始化(定义), 需要在类外定义初始化, 否则会报错

但是非静态成员可以在类中初始化(定义), 为什么？

**解释**：

编译器在创建对象时，会隐式调用构造函数，背后实际是对该对象内存调用一个以 `this` 为参数的函数，构造函数中自动插入基类构造、成员变量初始化、成员对象构造等代码逻辑，以确保对象完整初始化。

```c++
A obj;	//分配内存空间，然后隐式调用构造函数
```

根据以上内容, 静态成员变量不能在类内定义并初始化（在 C++11 之前）, 主要是因为它们**不属于对象实例，而属于整个类本身**。相比之下，非静态成员变量的初始化是通过编译器**在构造函数中自动插入赋值语句**来完成的，属于对象生命周期的一部分。而静态成员变量不会也不应该出现在构造函数中，编译器也不会将其初始化代码插入到对象构造过程中。因此，静态成员必须在类外显式定义和初始化，否则编译器无法分配内存并完成初始化，甚至会报错。

2. static成员描述的是整个类的属性, 是否加static需要从业务逻辑上去把握
3. static成员函数没有`this`指针, 这是显然的

## 2. Delegation(委托). Composition by reference

编译防火墙，pointer to implemention(PImpl),handle/body

![image-20250604231708745](c++面向对象高级编程上.assets/image-20250604231708745-17490502297959.png)

**Delegation 模式**：Composition by reference, Container 类中**持有对 Component 类的指针或引用**, 并通过它来实现具体功能, Container 本身只提供接口并将操作委托给 Component 实现。
 **生命周期**：Container 对象可以先于 Component 创建, 而 Component 对象通常在运行时按需构造, 因此两者生命周期相互独立、不同步。
